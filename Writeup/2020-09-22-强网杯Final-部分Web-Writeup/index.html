    强网杯Final 部分Web Writeup

    IEU

    这题非常非常的easy，本想是弱密码进入后台，然后读取shiro key文件，结果上台演示时发现直接时原生的shirokey，直接一把梭。

    这题显然不只是一个问题，我们可以发现如下几个漏洞

    在Shiro版本<1.6.0或可以登入后台时可以做到任意文件读取

    在com/ruoyi/web/controller/common/CommonController.java中

        @GetMapping("/common/download/resource")
    public void resourceDownload(String resource, HttpServletRequest request, HttpServletResponse response)
            throws Exception
    {
        // 本地资源路径
        String localPath = Global.getProfile();
        // 数据库资源地址
        String downloadPath = localPath + StringUtils.substringAfter(resource, Constants.RESOURCE_PREFIX);
        // 下载名称
        String downloadName = StringUtils.substringAfterLast(downloadPath, "/");

        response.setContentType(MediaType.APPLICATION_OCTET_STREAM_VALUE);
        FileUtils.setAttachmentResponseHeader(response, downloadName);

        FileUtils.writeBytes(downloadPath, response.getOutputStream());
    }

    由于public static final String RESOURCE_PREFIX = "/profile";我们可以构造resource=/profile/../../../../../../../etc/passwd读取文件

    在Shiro小于1.6.0时，由于其对静态资源的ShiroFilter为

    filterChainDefinitionMap.put("/css/**", "anon");

    所以我们可以构造/css/..;/<PATH>这样的访问路径进行访问

    在已知ShiroKey时可以利用Shiro反序列化进行RCE

    具体原因为在com/ruoyi/web/controller/system/SysLoginController.java中

        @PostMapping("/login")
    @ResponseBody
    public AjaxResult ajaxLogin(String username, String password, Boolean rememberMe)
    {
        UsernamePasswordToken token = new UsernamePasswordToken(username, password, rememberMe);
        Subject subject = SecurityUtils.getSubject();
        try
        {
            subject.login(token);
            return success();
        }
        catch (AuthenticationException e)
        {
            String msg = "用户或密码错误";
            if (StringUtils.isNotEmpty(e.getMessage()))
            {
                msg = e.getMessage();
            }
            return error(msg);
        }
    }

    调用了Subject subject = SecurityUtils.getSubject();而这个造成了Shiro的引入，导致了反序列化的发生

    通过以上两个漏洞的结合可以造成在Shiro<1.6.0时做到RCE

    ENA

    这题之前在给TSCTF出题的时候审计过，当时由于觉得没有什么洞就放弃了，结果梅开二度这题有洞，比赛的时候我人都傻了（x。

    这题在标准环境下存在两个漏洞

    首页任意文件上传

    这个洞其实在rw台上环境并不存在，这里不详细说了，主要就是首页存在一个ajax上传点，可以上传php后缀名的文件。

    任意文件包含

    在Application/Home/Controller/VueController.class.php文件中存在

    class VueController extends HomeBaseController{

    /**
     * 拦截空方法 自动加载html
     * @param  string $methed_name 空方法
     */ 
    public function _empty($methed_name){
        $this->display($methed_name);
        exit(0);
    }
    .........
}

    这里的$methed_name时可控的，甚至可以直接加载绝对路径

    这里display中具体处理文件解析的部分在ThinkPHP/Library/Think/View.class.php

        public function parseTemplate($template = '')
    {
        if (is_file($template)) {
            return $template;
        }
        .......
    }

    如果$template文件存在，直接返回其文件名，则不进行相关文件名处理

    在ThinkPHP/Library/Think/View.class.php中我们发现

     public function fetch($templateFile = '', $content = '', $prefix = '')
    {
        if (empty($content)) {
            $templateFile = $this->parseTemplate($templateFile);
            // 模板文件不存在直接返回
            if (!is_file($templateFile)) {
                E(L('_TEMPLATE_NOT_EXIST_') . ':' . $templateFile);
            }

        } 
        .......
        // 页面缓存
        ob_start();
        ob_implicit_flush(0);
        if ('php' == strtolower(C('TMPL_ENGINE_TYPE'))) {
            // 使用PHP原生模板
            $_content = $content;
            // 模板阵列变量分解成为独立变量
            extract($this->tVar, EXTR_OVERWRITE);
            // 直接载入PHP模板
            empty($_content) ? include $templateFile : eval('?>' . $_content);
        }
        ........
        return $content;
    }

    我们可以看到这里直接将模板文件包含了进来，所以造成了任意文件包含。

    利用日志文件+任意文件包含造成RCE

    所以说通过包含任意文件我们可以造成RCE，问题是我们包含什么文件？开始的想法是直接包含上传的文件，后来因为无法上传而搁置。
